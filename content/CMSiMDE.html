CMSiMDE

CMSiMDE 參考資料: https://mde.tw/cmstemplate

每一位工程師都需要一個 World Wide Web Page, 帶有分門別類的標題與內容, 能夠紀錄日誌, 可以直接在瀏覽器上進行簡報, 無論是在廣域網路, 區域網路或單一電腦上.

CMSiMDE 就是因應機械設計工程師需要建立上述

網頁 (Web Page)

簡報 (Web Presentation)

網誌 (Web Blog)

而利用 Flask 網路框架所建立的一套系統.

CMSiMDE 的網頁系統分為動態與靜態等兩部分, 動態網頁主要用於編輯各分頁內容, 而靜態網頁則用於展示.

CMSiMDE 架構

CMSiMDE 的倉儲內容分為用戶資料與程式引擎等兩個部分, 用戶資料存於 CMSMDE 倉儲中的 up_dir, 而其餘部分則屬於內容管理程式引擎.

使用時, 可以將 CMSiMDE 倉儲設為網站的子模組, 也可以直接取用 CMSiMDE 倉儲內容, 設為 cmsimde 子目錄.

用戶資料的靜態網頁位於 content 目錄, 網誌資料位於 blog 目錄, 而簡報資料則位於 reveal 目錄.

CMSiMDE 動態網站的內容存於 config/content.htm, 而網誌的 Leo Editor 專案檔位於 config/pelican.leo, 而簡報的 Leo Editor 專案檔則位於 config/reveal.leo.

在動態網站模式下, 經由瀏覽器上傳的檔案存於 downloads 目錄, 而上傳的影像檔案則存於 images 目錄.

CMSiMDE 的網誌系統採用 Pelican, 用來建立網誌的 Markdown 檔案位於 markdown 目錄, 而與 Pelican 網誌相關的 Plugin 則存於 plugin 目錄. 網誌樣式檔案則存於 theme 目錄.

templates 目錄存放的是延伸 CMSiMDE 程式時, 與 Flask 程式碼配合的頁面樣板檔案.

用戶資料中的 http-server.py 可以讓使用者在近端執行後, 啟動一個以 8444 作為 port number 的 https 近端 WWW 伺服器, 可以用來檢查所編輯完成的靜態網站內容.

當使用者要在近端啟動靜態網站時, 可以在命令列進入 cmsimde 目錄後, 以 python wsgi.py 啟動以 9443 作為埠號的 https 近端 WWW 伺服器, 可用於編輯動態與靜態網站各頁面內容.

CMSiMDE 的頁面分為三階, 在 config/content.htm 可以透過 H1, H2 與 H3 標註加以分頁.

CMSiMDE 的動態網站內容編輯告一段落後, 可以透過編輯表單最右方的 generate_page 按鈕轉為 content 目錄中的靜態分頁檔案.

CMSMDE 的網站標題與管理者密碼可以透過動態網站編輯表單中的 config 按鈕修改.

Gihut Pages 靜態網頁伺服

任何一個 Github 帳號都可以建立一個名稱為 "Github_帳號.github.io" 的倉儲, 並將CMSiMDE 靜態網站內容提交退送到此一倉儲, 一旦經由 Github Pages 轉換網站資料流程結束後, 就可以透過 https://Github_帳號.github.io 連結網站.

至於其他在 Github 的倉儲, 只要進入 Settings, 指定要設為 Github Pages 的對應分支, 就可以將該倉儲的網頁設為 https://Github_帳號.github.io/倉儲名稱.

Github Pages 將各倉儲資料轉為網頁的過程, 可以透過 Deploy Action 進行客製化定, 與 Github Pages 類似的 Gitlab Pages, 也允許使用者在倉儲中加入 .gitlab-ci.yml 設定檔案, 若以在 Gitlab Pages 配置 CMSiMDE submodule 為例, .gitlab-ci.yml 內容如下:

pages:
  stage: deploy
  script:
  - mkdir .public
  - cp -r * .public
  - mv .public public
  artifacts:
    paths:
    - public
  only:
  - master
variables:
  GIT_SUBMODULE_STRATEGY: recursive

Heroku 靜態網頁伺服

將 CMSMDE 靜態網站送到 Heroku 進行網頁伺服的方法是利用其內建使用 Apache 與 PHP 解譯系統處理 index.php 的特性, 只要將送到 Heroku 的 CMSiMDE 靜態網站資料中加入 index.php, 並將頁面導向 index.html 即可, 而 index.php 的內容如下:

<?php header( 'Location: /index.html' ) ;  ?>

根據此一架構配置完成的 CMSiMDE 網站: http://scrum1-cmsimde.herokuapp.com/
